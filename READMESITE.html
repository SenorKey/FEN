🔧 Server Troubleshooting Checklist

Step 1 — Check Apache
sudo systemctl status httpd
sudo systemctl start httpd
Step 2 — Check Firewall (look for "Apache Full" or ports 80/443)
sudo firewall-cmd --list-all

If missing:

sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

Note: Fedora uses firewalld, not ufw.

Step 3 — Confirm Public IP (update Namecheap A record if changed)
curl -4 ifconfig.me

Namecheap DNS: Host @ → Value: [your public IP] · TTL: Automatic

Step 4 — Confirm Local IP (for Xfinity port forwarding)
hostname -I

Should be something like 10.0.0.XXX. Forward ports 80 and 443 to this IP in the Xfinity app.

📦 Git Workflow

First-Time Setup (per machine)

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git clone https://github.com/SenorKey/your-repo.git
cd your-repo

Everyday Workflow

git pull origin main          # Always pull first
# make your changes...
git status                    # Check what changed
git add .                     # Stage all
git add index.html            # Stage a specific file
git commit -m "your message"
git push origin main

Pulling on the Fedora Server

cd /var/www/frontendneeded.com
git pull origin main

Making a New Repo

# 1. Create the repo on GitHub first, then:
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/SenorKey/your-repo.git
git branch -M main
git push -u origin main

# Fix wrong remote URL:
git remote set-url origin https://github.com/SenorKey/your-repo.git

Useful Git Commands

git log --oneline            # Recent commit history
git diff                     # Unstaged changes
git diff --staged            # Staged but uncommitted changes
git restore filename.html    # Discard changes to a file
git stash                    # Shelve uncommitted changes
git stash pop                # Re-apply stashed changes
git branch                   # List branches
git checkout -b new-branch   # Create and switch to new branch
git merge branch-name        # Merge branch into current
git remote -v                # Check which repo you're pointed at
git status                   # Branch and file status

🐧 Fedora / Linux Commands

Navigation

ls / ls -la                  # List files (with hidden + details)
cd /path/to/folder
cd ~                         # Home directory
cd ..                        # Up one level

File Management

touch filename.txt
cp file.txt /path/to/dest/
mv file.txt /path/to/dest/
rm filename.txt

Viewing & Editing

sudo nano /etc/somefile

Permissions
